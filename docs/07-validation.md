# 07 — Post-install validation

After running `setup.sh` and rebooting (if section 02 installed a new kernel), verify everything's working.

## Quick check

```bash
zenbook-validate
```

Generated by section 10, available at `~/.local/bin/zenbook-validate`. Add `~/.local/bin` to your `PATH` if it isn't already (Ubuntu does this by default in `~/.profile`).

The output groups checks into:

- **OS / Kernel** — should show Ubuntu 24.04 and `6.17.0-20-generic`
- **amdxdna NPU** — four detection methods, all should pass
- **GPU / Vulkan** — AMD Radeon detected
- **Build tools** — gcc, g++, cmake, make, git, pkg-config
- **Languages** — whatever `mise` has installed for you, plus system fallbacks
- **Container** — Docker + Compose v2
- **AI stack** — only relevant if you used `--with-ai-stack`
- **Apps stack** — only relevant if you used `--with-apps`
- **Flatpak apps** — only relevant if you used `--with-flatpak`
- **Gaming** — only relevant if you used `--with-gaming`
- **Kernel hold status** — must show `linux-image-generic` is held
- **Storage** — `df -h` for sanity

## Manual smoke tests

A few that aren't covered by the script:

### Kernel + NPU

```bash
uname -r                                   # 6.17.0-20-generic
sudo dmesg | grep -i amdxdna | head        # no "SVA bind device failed"
ls -la /dev/accel/                         # accel0 (or similar) exists
```

### Wayland

```bash
echo "$XDG_SESSION_TYPE"                   # wayland
loginctl show-session "$(loginctl | grep $USER | awk '{print $1}')" -p Type   # Type=wayland
```

If you get `x11`, log out, click the gear icon at the login screen, pick "Ubuntu (Wayland)".

### Vulkan

```bash
vulkaninfo --summary | grep deviceName
# Expect: AMD Radeon Graphics (or similar) for RADV
```

### VA-API (hardware video decode)

```bash
vainfo
# Expect a long codec list including H.264, H.265, AV1
```

### Docker

```bash
docker run --rm hello-world
docker compose version
```

If you get `permission denied`, you haven't logged out + in since being added to the `docker` group. Either log out and back in, or run `newgrp docker` in the current shell.

### `mise`

```bash
mise --version
which python3                              # should be /home/USER/.local/share/mise/installs/python/.../bin/python3 if you've added Python
mise ls
```

### LaTeX (if you installed `texlive-full` manually)

```bash
echo '\documentclass{article}\begin{document}Hello\end{document}' > /tmp/test.tex
cd /tmp && pdflatex test.tex
```

## What "good" looks like

A healthy install on a fresh Zenbook S16 should:

- Boot in <10 s on NVMe
- Show `6.17.0-20-generic` after reboot
- Have `amdxdna` listed in `lsmod`
- Run `vulkaninfo --summary` cleanly with `AMD Radeon Graphics`
- Pass `docker run hello-world`
- `~/.cache/zenbook-s16-setup/setup-*.log` should not contain `[ERROR]` lines (warnings are fine)

If any of those fail, see [`08-troubleshooting.md`](08-troubleshooting.md).
